# 🏛️ AI 奧勒岡 2v2 雙人辯論模擬器 (AI Debate Simulator)

本專案是一個基於大語言模型（如 Google Gemma 4 及 Gemini 系列）的自動化辯論模擬沙盒。系統採用**奧勒岡雙人 (2v2) 辯論賽制**流程，在每次辯論中分配不同的思想流派（理性、感性、保守、改革）角色進行立論、攻防詰問、與總結陳詞。

專案提供了一個直觀、精美的 **Streamlit 網頁前端介面**，讓使用者能夠快速配置 API Key、自訂或選擇多樣化議題，並即時觀看 AI 辯士之間的精彩對話。

---

## 🌟 核心特色
1. **多重人格特質 (Debate Personalities)**:
   - **🔵 理性派**: 冷靜分析、講求邏輯與數據，善用因果推論，避開情緒性字眼。
   - **🔴 感性派**: 高同理心，關注人類情感、生命經驗與價值衝突，善用故事說服。
   - **🟢 保守派**: 重視秩序與穩定，強調風險控制、傳統智慧與漸進式改革。
   - **🟣 改革派**: 批判性思維強，關注社會公平與未來趨勢，提出創新解決方案。
2. **奧勒岡 2v2 線性賽制**:
   - **第一階段**: 正方一辯申論
   - **第二階段**: 反方一辯質詢 (對正方一辯進行詰問)
   - **第三階段**: 反方二辯申論
   - **第四階段**: 正方二辯質詢 (對反方二辯進行詰問)
   - **第五階段**: 反方一辯結辯 (總結漏洞並重申立場)
   - **第六階段**: 正方一辯結辯 (總結漏洞並重申立場)
3. **智慧角色分配**:
   - 預設模式下，系統會依議題特性優化角色位置（例如討論「穩定與自由」時，將保守派分至正方、改革派分至反方）。
4. **防呆與清洗機制**:
   - 自動過濾推理模型輸出的內心思考 `<think> ... </think>` 標籤，確保最終發言之簡潔。
   - 內建發言冷卻機制（延遲），有效防止 Rate Limit (請求限制) 與 Timeout 問題。
5. **完整戰局整理與匯出**:
   - 側邊即時顯示 **Flow Sheet (爭點整理)**。
   - 辯論結束後可一鍵下載 Markdown 格式的完整發言記錄。

## 🛠️ 開始使用 (Local Setup)

### 1. 安裝環境與依賴套件
確保您已安裝 Python 3.10+，並在專案根目錄下執行以下指令安裝依賴包：
```bash
pip install -r requirements.txt
```

### 2. 配置 API Key
您可以使用以下兩種方式之一來設定您的 Google Gemini API Key：
- **方式 A**: 在專案根目錄下建立 `.env` 檔案並填入金鑰：
  ```text
  GOOGLE_API_KEY=your_gemini_api_key_here
  ```
- **方式 B**: 啟動網頁後，直接在左側邊欄輸入 API 金鑰（輸入的金鑰將優先使用且不對外洩漏）。

### 3. 啟動 Streamlit 網頁
在專案根目錄執行以下指令啟動前端網頁：
```bash
streamlit run app.py
```
啟動後，瀏覽器會自動開啟本地網頁（一般為 `http://localhost:8501`）。

## 📝 遇到的問題與解決方案
1. **API Rate Limit (頻率限制)**:
   - **對策**: 網頁端內建發言冷卻延遲控制（可在側邊欄調整 2~20 秒），防止模型請求過於頻繁。
2. **思考過程外洩**:
   - **對策**: `app.py` 中寫入了正則表達式清洗器，能強制剔除所有 `<think>` 段落，保持發言純淨。
3. **檔案讀取失敗**:
   - **對策**: 採用智慧多路徑尋找機制，不論在專案根目錄或子目錄啟動，皆能自動正確認讀 `角色卡與辯題/` 下的所有 Markdown 設定。
